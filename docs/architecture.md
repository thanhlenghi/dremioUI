# Architecture

The browser never calls Dremio or OpenAI directly. It talks to the FastAPI backend through `/api`.

## Runtime

- React frontend served by Nginx.
- FastAPI backend for Dremio proxying, session handling, SQL submission, and Q&A policy.
- Redis stores session-only Dremio tokens keyed by an HTTP-only cookie session id.
- OpenAI is used only by the backend and only receives metadata, SQL text, job status, durations, errors, and counts.

## V1 Boundaries

- Admin surfaces for users, roles, RBAC, and engines are read-only.
- SQL generated by Q&A is a draft. The user must manually run it.
- Query result rows are not sent to OpenAI by default.
- Job history is loaded through a bounded recent window using Dremio SQL over system job metadata.

## Deferred

- TLS/reverse-proxy hardening.
- Durable app database, audit logs, saved views, and indexed historical job search.
- Mutations for RBAC, users, roles, sources, or engines.
